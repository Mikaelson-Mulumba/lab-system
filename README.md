🧪 Offline Lab System (Flutter + SQLite)
Overview
The Offline Lab System is a mobile‑first application designed to manage laboratory operations in environments with limited or unreliable internet connectivity. Built with Flutter for cross‑platform deployment and SQLite for secure offline storage, it empowers medical staff to record, process, and analyze lab data seamlessly. Once connectivity is restored, the system synchronizes data securely to the cloud.

✨ Key Features
Offline‑First Architecture  
Operates fully without internet access, ensuring continuity in rural or resource‑constrained labs.

SQLite Local Database  
Reliable, lightweight storage for patient records, test results, and inventory — optimized for mobile devices.

Data Synchronization  
Automatic sync with central servers when connectivity is available, preventing data loss and duplication.

Patient & Sample Management  
Streamlined workflows for registering patients, tracking samples, and recording test results.



Secure & Compliant  
Encrypted local storage, role‑based access, and audit trails to protect sensitive medical data.

🛠️ Tech Stack
Frontend: Flutter (cross‑platform: Android, iOS, Web)

Database: SQLite (offline storage)



Version Control: GitHub with clean baselines and CI/CD pipelines

🌍 Impact

bash

bash

Deploy to Android/iOS devices or Web.
